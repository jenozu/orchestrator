# Naming Conventions

## Files and Directories

- Use lowercase with underscores for Python files: `my_module.py`
- Use kebab-case for Markdown files: `my-document.md`
- Use descriptive names that indicate purpose

## Variables and Functions

- Use snake_case for variables and functions: `user_name`, `get_user_data()`
- Use descriptive names: `total_count` not `tc`
- Constants should be UPPER_CASE: `MAX_RETRIES`

## Classes

- Use PascalCase: `UserManager`, `DataProcessor`
- Use nouns that describe the class purpose

## Modules and Packages

- Use lowercase with underscores: `data_processing`, `api_client`
- Keep names short but descriptive
